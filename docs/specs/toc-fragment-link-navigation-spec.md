# In-Article Fragment Link Navigation — Technical Specification

Status: Draft for implementation
Date: 2026-03-22

## 1. Purpose

Enable same-page anchor links (e.g. table-of-contents links) to work in the reader WebView.
Currently, tapping a TOC link does nothing. This spec describes why that happens and how to fix it.

## 2. Background

Article HTML returned by the Readeck API can include a table of contents generated by the
original site. Anchor links in that TOC have `href` values that are fragment-only references,
for example:

```html
<a href="#uY.XlwQ.Seek_Using_-ss_Parameter">Seek Using -ss Parameter</a>
```

The target heading carries a matching `id` attribute:

```html
<h2 id="uY.XlwQ.Seek_Using_-ss_Parameter">Seek Using -ss Parameter</h2>
```

Readeck's sanitiser prefixes IDs and fragment hrefs with a per-article token (e.g. `uY.XlwQ.`)
to prevent ID collisions when multiple articles are embedded in one page. The prefix does not
affect correctness on Android because each article is loaded in its own WebView.

## 3. Root Cause

The reader WebView is not itself scrollable. It is rendered at full content height inside a
Compose `Column` that carries `.verticalScroll(scrollState)`. All scrolling is driven by
Compose's `ScrollState`; the WebView has no internal scroll offset of its own.

When the user taps a fragment link the Android WebView recognises it as a same-page anchor
navigation and handles it internally — `shouldOverrideUrlLoading` is **not** called. The WebView
attempts to scroll its own viewport to the target element, but because its scroll offset is
always zero (Compose owns the scroll), the attempt is a no-op and nothing visible happens.

This is identical to the problem solved for annotation jump-to, which uses
`scrollState.animateScrollTo()` after querying the target element's absolute position via
JavaScript.

## 4. Proposed Solution

### 4.1 Overview

Inject a JavaScript click interceptor that captures taps on fragment links, resolves the target
element's absolute Y position, and reports it to Kotlin via a JS bridge. Kotlin then drives
`scrollState.animateScrollTo()` — the same path used by annotation scrolling.

No Compose layout changes are required. The `scrollState` is already threaded into
`BookmarkDetailArticle` and a coroutine scope for animation is already present.

### 4.2 JavaScript interceptor

Add a new companion function `injectFragmentLinkInterceptor()` to `WebViewImageBridge`
(or a dedicated `WebViewTocBridge` if preferred for separation of concerns).

The injected script:

1. Queries all `<a>` elements whose `href` attribute starts with `#`.
2. Attaches a `click` event listener to each with `capture = true`.
3. On click, calls `e.preventDefault()` and `e.stopPropagation()` to suppress the WebView's
   own fragment navigation attempt.
4. Resolves the target element using an **attribute selector**:
   ```js
   document.querySelector('[id="' + targetId + '"]')
   ```
   CSS selector syntax (e.g. `#uY.XlwQ.foo`) must be avoided because dots are interpreted as
   class selectors. The attribute selector form is unambiguous. `getElementById` would also work,
   but the attribute selector is more explicit about the intent.
5. Computes the element's absolute Y position:
   ```js
   var rect = el.getBoundingClientRect();
   var absoluteY = Math.round(rect.top + window.scrollY);
   ```
   Because the WebView scroll offset is always zero, `window.scrollY` is always 0, but
   including it keeps the formula consistent with the annotation bridge.
6. Calls the native bridge method, e.g.:
   ```js
   MyDeckTocBridge.onFragmentLinkTapped(absoluteY);
   ```

**Fragment extraction** from the `href` attribute:

```js
var href = a.getAttribute('href');         // e.g. "#uY.XlwQ.Seek_Using_-ss_Parameter"
var targetId = href.slice(1);              // strip leading '#'
```

**Security note:** The fragment value is not embedded back into a JS string after this point;
it is only used as a DOM query argument within the same script invocation, so injection risk
is negligible. The absoluteY integer passed to the bridge is a number, not a string.

### 4.3 Native bridge

Add a new `@JavascriptInterface` method (either on a new `WebViewTocBridge` class or on an
existing bridge):

```kotlin
@JavascriptInterface
fun onFragmentLinkTapped(absoluteY: Int) {
    Handler(Looper.getMainLooper()).post {
        onFragmentScroll(absoluteY)
    }
}
```

`onFragmentScroll` is a callback lambda threaded through to `BookmarkDetailArticle`, analogous
to `onImageTapped`.

### 4.4 Scroll execution

In `BookmarkDetailArticle` (or its parent screen), handle the callback in a `LaunchedEffect`
or `rememberCoroutineScope` block:

```kotlin
coroutineScope.launch {
    scrollState.animateScrollTo(absoluteY)
}
```

The value from the bridge is the target element's top edge in document pixels. Optionally
subtract a small offset (e.g. the toolbar height) to leave visual breathing room above the
heading, consistent with how annotation scrolling centres the target.

### 4.5 Injection point

Call `injectFragmentLinkInterceptor()` in `applyReaderEnhancements()` alongside the existing
image and annotation interceptors, so it runs on both `onPageCommitVisible` and `onPageFinished`.

## 5. Scope Constraints

- **Article content only.** Fragment links in video or photo content are not expected; no
  changes needed to those content-type paths.
- **Offline content.** Offline articles use the same WebView setup and scroll model. The fix
  applies equally without any offline-specific changes.
- **External links are unaffected.** `shouldOverrideUrlLoading` continues to handle all
  non-fragment link clicks and open them in Chrome Custom Tabs.
- **No layout changes.** The Compose hierarchy, `ScrollState` wiring, and WebView sizing are
  unchanged.

## 6. Files Affected

| File | Change |
|------|--------|
| `WebViewImageBridge.kt` (or new `WebViewTocBridge.kt`) | Add `injectFragmentLinkInterceptor()` JS and `@JavascriptInterface` bridge method |
| `BookmarkDetailWebViews.kt` | Register bridge on WebView; call interceptor in `applyReaderEnhancements()`; wire `onFragmentScroll` callback |
| `BookmarkDetailScreen.kt` | Handle scroll callback and call `scrollState.animateScrollTo()` |

## 7. Out of Scope

- Smooth-scroll animation style (uses whatever `animateScrollTo` default provides).
- Back-navigation after following a TOC link (not required; the user can scroll back manually).
- Highlighting the active TOC entry.
- Multi-column or paginated reading modes (not implemented in this app).
